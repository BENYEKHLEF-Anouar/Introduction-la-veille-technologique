# Introduction : Contexte de la Veille

Google Antigravity est une plateforme de développement **agentique** lancée le **18 novembre 2025**, en parallèle du modèle Gemini 3.

- C'est un **IDE gratuit** (en preview publique) pour Windows, macOS et Linux.
- Basé sur une fork de VS Code, mais repensé autour des **agents IA autonomes**.
- Objectif : Passer d'un assistant de code (comme Copilot) à un **partenaire actif** qui planifie, code, teste et vérifie.

> "Antigravity transforme l'IA d'un outil passif en un partenaire autonome." – Google Developers Blog

---

# Qu'est-ce que Google Antigravity ?

### Paradigme "Agent-First"
- Les agents IA ne se contentent pas de suggérer du code : ils **planifient**, **exécutent** et **valident** des tâches complexes.
- Accès direct à l'éditeur, au terminal et au navigateur intégré.
- Deux vues principales :
  - **Editor View** : IDE classique avec aide IA (pour le travail manuel).
  - **Manager View** : "Mission Control" pour orchestrer plusieurs agents en parallèle.

### Modèles sous-jacents
- Principalement **Gemini 3 Pro** (le plus avancé de Google).
- Support aussi Claude Sonnet 4.5 et modèles open-source.

# Démonstration Concrète : Notes App

J'ai utilisé Antigravity pour créer une **application complète de prise de notes** en **moins de 5 minutes** :

- **Technologies** : HTML, CSS (Glassmorphism premium), JavaScript Vanilla.
- **Fonctionnalités** : Ajout/suppression de notes, stockage local (localStorage).
- **Rôle de l'agent** : 
  - Génération complète du design et du CSS (aucune ligne écrite manuellement).
  - Implémentation de la logique JS.
  - Test via navigateur intégré.

> Résultat : Une app fonctionnelle et esthétique, prête à être étendue.

# Avantages et Limitations

### Avantages
- **Vitesse x10** : Idéal pour les prototypes et les projets scolaires.
- **Focus sur la logique métier** : Moins de temps sur le CSS/boilerplate.
- **Apprentissage** : Voir comment un agent "raisonne" et structure un projet.

### Limitations (à surveiller)
- Qualité du code : Toujours reviewer (risque d'hallucinations ou de code non optimal).
- Sécurité : Permissions terminal/navigateur à configurer soigneusement.
- Preview publique : Limites de taux et fonctionnalités en évolution.
